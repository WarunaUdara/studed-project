# Glossary

> [!info] Purpose
> A centralized dictionary of terms used across the StudEd project. Keep this updated as new concepts are introduced.

## A

**AI Assistant**
: The integrated LLM-powered tool inside the [[MDX Editor]] that helps educators generate and improve content. See [[AI Integration]].

**Attempt**
: A single submission of the [[Evaluate Component|Evaluate]] phase by a student. Tracked in [[Progress Tracking]] and limited by [[Reattempt Mechanics]].

## B

**Block**
: The atomic unit of content in the [[MDX Editor]]. Can be a [[Learn Component|Learn block]] (text, image, audio) or an [[Evaluate Component|Evaluate block]] (MCQ, fill-in-blank, drag-and-drop).

**Puck**
: The selected open-source visual editor library for StudEd. See [[Puck Research]].

## C

**ChemViz Block**
: A [[Learn Component]] block that renders interactive 3D molecular visualizations using [[3Dmol.js Integration|3Dmol.js]]. Used for chemistry and biology content.

**Course**
: The top-level container in the [[Course-Lesson-Wave-Hierarchy|content hierarchy]]. A Course contains multiple Lessons (e.g., "Mathematics - O/L").

**Creator**
: See **Educator**.

## D

**Drag-and-Drop**
: Both an interaction pattern in the [[MDX Editor]] (reordering blocks) and a question type in the [[Evaluate Component]] (matching/ordering exercise).

## E

**Educator**
: A user role responsible for creating Courses, Lessons, and Waves. Uses the [[Educator Dashboard]] and [[MDX Editor]].

**Educator AI Chat Interface**
: A conversational panel where educators upload photos and use natural language to have AI generate complete Wave layouts. See [[Educator AI Chat Interface]].

**ElecSim Block**
: A [[Learn Component]] block that renders interactive electronics circuit schematics using [[tscircuit Integration|tscircuit]]. Used for physics and electronics content.

**Evaluate**
: The assessment phase of a [[Wave]], consisting of quizzes and exercises. See [[Evaluate Component]].

**Expert**
: The highest [[Proficiency System|proficiency level]], achieved when all Waves in a Lesson are completed with a 100% average score.

**XP (Experience Points)**
: The gamification currency awarded for completing Waves and achieving milestones. See [[XP-System]].

## F

**Fill-in-the-Blank**
: An [[Evaluate Component]] question type where students type missing words into a sentence.

## G

**Gamification**
: The use of game-like mechanics (XP, leaderboards, badges) to motivate learning. See [[XP-System]], [[Leaderboards]], [[Proficiency System]].

**Gemini**
: **Google Gemini 3.5 Flash**, the primary LLM used for Sinhala language tasks (OCR, text generation, translation) in the [[AI Content Generation Service]].

**Grade**
: The academic level of a student (Grade 1–11, O/L, A/L). Used for content targeting and leaderboards.

## H

**Hierarchy**
: StudEd's strict three-tier content structure: **Course** → **Lesson** → **Wave**. See [[Course-Lesson-Wave-Hierarchy]].

## L

**Leaderboard**
: A ranked list of students by total [[XP-System|XP]], scoped globally, by Course, by Grade, or weekly. See [[Leaderboards]].

**Learn**
: The educational phase of a [[Wave]], delivering content through text, images, audio, and graphics. See [[Learn Component]].

**Lesson**
: The second tier in the content hierarchy. A Lesson contains multiple Waves and belongs to a Course.

**LLM**
: Large Language Model. The AI technology (e.g., Gemini, Qwen, DeepSeek) powering the [[AI Integration|AI assistant]].

## M

**Manim**
: A Python library for creating mathematical animations. Used in the [[Math-To-Manim Integration]] to generate [[MathViz Block|MathViz]] animations.

**MathViz Block**
: A [[Learn Component]] block that renders mathematical animations generated by [[Math-To-Manim Integration|Math-To-Manim]]. Used for geometry, calculus, and algebra visualizations.

**Matter.js**
: A 2D rigid body physics engine for the web. Used in the [[Matter.js Integration]] to create [[MechSim Block|MechSim]] interactive physics simulations.

**MCQ**
: Multiple Choice Question. An [[Evaluate Component]] block with selectable options. See [[Evaluate Component#Multiple Choice Question (MCQ)]].

**MDX Editor**
: The drag-and-drop content creation tool used by educators. See [[MDX Editor]].

**MechSim Block**
: A [[Learn Component]] block that renders interactive 2D physics simulations using [[Matter.js Integration|Matter.js]]. Used for mechanics, pendulums, collisions, and projectile motion.

## O

**O/L**
: Ordinary Level. A Sri Lankan national exam taken at Grade 11.

## P

**Passing Threshold**
: The minimum percentage score required to mark a [[Wave]] as completed. Set by the educator.

**Proficiency**
: A status indicating deep mastery of a Lesson (all Waves completed with an average score >= 80%). See [[Proficiency System]].

**Qwen**
: Alibaba's **Qwen 2.5 72B** open-source LLM, used for general pedagogy and reasoning tasks in the [[AI Content Generation Service]].

**DeepSeek**
: **DeepSeek-Coder** and **DeepSeek-V3** open-source models used for code generation and advanced reasoning in the [[AI Content Generation Service]].

## R

**Reattempt**
: A retry of a [[Wave]]'s Evaluate phase, allowed up to a capped limit. See [[Reattempt Mechanics]].

## S

**Sinhala**
: The primary language (besides English) that StudEd must support. See [[Sinhala Language Support]].

**Student**
: The primary end-user role. Students consume content, earn XP, and compete on leaderboards.

**Subscription**
: The paid access model. Students must have an active subscription to unlock Courses. See [[Monetization Strategy]].

## T

**Tier**
: A subscription level (Basic, Standard, Premium, School License) that determines access scope. See [[Monetization Strategy#Subscription Tiers (Proposed)]].

**tscircuit**
: An open-source TypeScript/React library for designing and visualizing electronic circuits. Used in the [[tscircuit Integration]] to create [[ElecSim Block|ElecSim]] interactive circuit simulations.

## W

**Wave**
: The core interactive unit of StudEd. A "Level" containing a [[Learn Component|Learn]] phase and an [[Evaluate Component|Evaluate]] phase. See [[Wave Anatomy]].

**Webhook**
: A mechanism by which payment providers (e.g., PayHere) notify StudEd of payment events. See [[Payment Integration]].
